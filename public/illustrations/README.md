# 挿絵 (illustrations) ディレクトリ

教材ページのカバー画像・本文背景に使う挿絵を `<collection>/<slug>/sceneN.webp` の構成で配置しています。

すべて教材 `.md` の frontmatter `cover_image:` フィールドから参照され、Astro のビルド時に `/illustrations/...` として静的配信されます。

## ファイル仕様

| 項目         | 値                                                            |
| ------------ | ------------------------------------------------------------- |
| フォーマット | WebP（VP8, lossy）                                            |
| 解像度       | 横 1280 px、縦は元画像に応じて（多くは 1280×721）             |
| 品質         | `cwebp -q 82`                                                 |
| サイズ目安   | 1 枚あたり 100–280 KB                                         |
| アスペクト   | 16:9 横長 — Lesson ページのカバー領域と OG カードに合うため   |
| 文字         | 画像内に文字は描き込まない（多言語対応・著作権リスク回避）    |

## 生成パイプライン

挿絵は ChatGPT (DALL-E / GPT Image-1) を Mac の Google Chrome 上で操作して生成し、ローカルでバッチ変換しています。フローは下記の通り：

1. **プロンプト**：Lesson の主題に沿った日本語のプロンプトを ChatGPT に投げる
   - スタイル：教育教材用、落ち着いた色調、写実 or 浮世絵風 or 水彩、横長 16:9、文字なし
   - 例：「DALL-E ですぐ画像を生成してください。『江戸時代の寺子屋で算盤を弾く子どもたち』。墨絵・浮世絵風、横長 16:9、文字なし、教育教材用の落ち着いた美しさで。」
2. **取得**：生成画像を Canvas → Blob → 合成 `<a download>` で `~/Downloads/*.png` に保存
3. **WebP 変換**：`cwebp -q 82 -resize 1280 0 input.png -o public/illustrations/<collection>/<slug>/scene1.webp`
4. **frontmatter 更新**：教材 `.md` の `keywords:` 直後に下記を挿入

   ```yaml
   cover_image: /illustrations/<collection>/<slug>/scene1.webp
   ```

5. **ビルド検証**：`npm run build`
6. **コミット**：`git add` → `git commit`（バッチ単位で）

## 追加するときの規約

1. **1 教材 1 カバー** を最低保証 — `scene1.webp` のみで OK。本文中に複数シーンを出したい場合のみ `scene2.webp` 以降を追加
2. **ファイル名は `sceneN.webp`** — 番号は本文出現順
3. **WebP 以外（PNG / JPG）はコミットしない** — Astro のアセット最適化と Cloudflare の帯域を尊重
4. **絵にテキストを焼き込まない** — i18n 対応と、生成 AI の文字レンダリング品質の問題を回避
5. **生成元プロンプトのログ** — 復元可能性のため、コミットメッセージにプロンプト要旨を残すと親切
6. **ライセンス** — 生成画像は CC-BY-4.0 として配布（編集物扱い）

## 既知の制約

- DALL-E は日本史・古典の人物像で時代考証を間違えがち。歴史教材では「学生服を着た武士」など出ない最低限のチェックを目視で行う
- 一度の生成で 60–90 秒かかる。バッチ処理時はタイムアウトに注意
- ChatGPT は同じ会話に 30 枚以上ためると重くなる。30 枚ごとに新規チャットへ移行する
