## サマリー

### 全体評価

全26教材は、編集方針 (`docs/editorial-policy.md`) およびスキーマ (`src/content.config.ts`) に概ね準拠しており、内容の質は総じて高い。とくにテンプレート教材である `tebukuro-wo-kaini.md` の完成度は高く、それを基準として比較すると、他教材にも体系的な改善余地が見える。最大の課題は **(1) 学年配当外の難読漢字へのルビ抜け（とくに kokugo3、chugaku1）** と **(2) `curriculum_items` の「コード」表記の不統一・指導要領との不整合**。

### 共通の傾向

**良い点**
- 出典・底本記載は全ファイル整っており、`source.url` / `source.base` が省かれているのは古典（公有古典、政府文書）のみで、その場合の `base` 表記も方針通り。
- glossary の文体は学校段に応じて差別化されている（小3は「むかし〜」、高1は学術語）。
- 描写・引用ブロック（`blockquote class="song"`）の使い分けが一貫している。
- ライセンス情報（`license_editorial`, `license_original`）は全教材で正しく設定されている。

**改善点（パターン化されたもの）**
1. **ルビ抜け**：小3・中1の文学教材で、学年配当外の漢字に振り仮名が振られていない箇所が散見される（特に kokugo3/issun-boshi.md、chumon-no-oi-ryoriten.md）。
2. **curriculum_items の表記揺れ**：（a）「C(1)エ」「C(1)イ」のようなコード形式と、「中1 第1分野」「英語コミュニケーションI」のような領域名形式が混在。schema 上は両方とも `string` で許容されるが、学校段ごとに統一すべき。（b）`curriculum-mapping.md` には小3用しかマッピングが書かれていないので、中・高用のドキュメントが手薄。
3. **「先生」「下女」など現代では再考すべき表現**：作品保存の観点で残すべきものではあるが、編集方針 §4 が指す「差別的表現」への注釈が一切付されていない教材がある（注文の多い料理店「下女」、伊勢物語「お嬢中」など）。
4. **kotokoeigo1 の glossary**：英語の語彙について、学習者の語彙水準（高1）との整合性が薄い箇所がある（"merchant" は中学範囲、"ferryman" は文脈付与のみで OK 等）。
5. **`questions` の問い方の偏り**：「本文の言葉を引いて答えましょう」型と「あなたの考えを書きましょう」型はバランス良いが、「他者の意見を聞いて考える」「他作品との比較」型が少ない（curriculum_items の C(1)オ「共有」を意識すべき）。
6. **HTML の閉じ忘れ・破綻**：構造的な不正はなかった（よい）。
7. **明らかなタイポ・誤字**：ほぼなし。ただし `chugakurika1/konpeito.md` のタイトルが frontmatter で「金米糖」、ファイル名は「konpeito」と微妙な不一致あり。

## 横断的な推奨事項

### 全教材で改善すべきパターン

1. **glossary 内の HTML 制御**: 一部教材で `<strong>` を含む `meaning` がある。glossary は plain text に揃えるルールを編集方針に明文化することを推奨。

2. **`<blockquote class="song">` 内の `<strong>` の扱い**: 原文引用に強調を加えるのは編集としては許容範囲だが、「これは編集者の加筆である」と読者に分かる仕組み（例：背景色を変える、注を入れる）が欲しい。

3. **`curriculum_items` のコード命名規則の統一**: 
   - 小・中・高で形式が異なるのは仕方ないが、`curriculum-mapping.md` を中・高用にも拡張し、コードの ground truth を提供する。
   - 現状、`code: "C(1)エ"` `code: "中1 第1分野"` `code: "英語コミュニケーションI"` のような揺れがある。

4. **「差別的表現」への注釈の有無**: 編集方針 §4 にある3つの選択肢（そのまま掲載＋注釈／改稿／不採用）が、現状どの教材も「そのまま掲載」になっているが、注釈の有無が不統一。注釈ポリシーを明文化すべき。

5. **glossary の term と本文の整合性**: 本文では「<ruby>X<rt>y</rt></ruby>」、glossary では「X（y）」など、表記揺れが多い。 → **提案**: term は「漢字（読み）」形式に統一、本文と glossary を機械的に照合可能にする。

6. **`questions` バランスの見直し**: 「本文の言葉を引いて答える」「自分の考えを書く」型は十分、「他者と共有する／意見を聞き合う」型が少ない。指導要領（小C(1)オ、中C(1)オ、高B(1)オ）に対応する問いを意識的に1問入れる方針が望ましい。

### スキーマの拡張提案

1. **`glossary` の構造拡張**：
   ```ts
   glossary: z.array(z.object({
     term: z.string(),
     reading: z.string().optional(),  // 読み方を独立フィールドに
     meaning: z.string(),
     category: z.enum(['vocabulary', 'concept', 'person', 'place', 'work']).optional(),
   }))
   ```

2. **`questions` の構造拡張**：
   ```ts
   questions: z.array(z.object({
     prompt: z.string(),
     type: z.enum(['factual', 'interpretive', 'personal', 'comparative', 'shared']).optional(),
     curriculum_item: z.string().optional(),  // どの指導事項に対応するか
   }).or(z.string()))  // 後方互換
   ```

3. **`editor_notes` フィールドの追加**：
   ```ts
   editor_notes: z.array(z.object({
     section: z.string(),  // 例: "本文 §四", "glossary"
     note: z.string(),
   })).default([])
   ```
   これにより、編集者が「差別語を残した理由」「旧字を新字にした箇所」などをメタデータとして記録できる。

4. **`reading_level` フィールド**：
   ```ts
   reading_level: z.object({
     furigana_density: z.enum(['high', 'medium', 'low']).optional(),
     classical_grammar: z.boolean().default(false),
   }).optional()
   ```
   ルビ密度や文語の有無を明示し、ページ側で UI 表示を調整できる。

5. **`curriculum-mapping.md` の中学・高校・各教科版作成**：
   - 現状は小3用のみ。中学（国語・理科・社会）、高校（国語・英語）、小5算数用にも別ドキュメントを作成。
